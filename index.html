1. Blockchain Fundamentals (Q1-10)

This section covers the core building blocks of blockchain technology. You must know these definitions inside and out as they are the foundation for the entire subject.

1. What is a blockchain? Explain its core properties.

A blockchain is a decentralized, distributed, and immutable digital ledger.

  • Decentralization: No single entity controls the network; it's maintained by a network of computers called nodes.
  • Immutability: Data cannot be altered or deleted once recorded, secured by cryptographic hashes.
  • Transparency: All transactions are publicly visible on the network.
2. How does a blockchain differ from a traditional database?
  • Control: Blockchain is decentralized, while a traditional database is centralized.
  • Data Mutability: Blockchain data is immutable, whereas database data can be changed.
  • Consensus: Blockchains use consensus mechanisms for data validation, unlike databases with a central administrator.
3. What is a block and what components does it contain?

A block is a container for data. It contains:

  • Block Header: Metadata like timestamp and Merkle root.
  • Transactions: A list of verified transactions.
  • Hash: A unique fingerprint of the block's content.
  • Nonce: A random number used in mining to find a valid hash.
4. Explain the role of a cryptographic hash function in a blockchain.

A hash function takes input data and produces a unique, fixed-size output. Each block's hash is a cryptographic fingerprint of its content. To ensure immutability, each block's header includes the hash of the *previous* block, creating an unbreakable chain. Any change to a block's data changes its hash, breaking the chain.

5. What is a Merkle tree and why is it important?

A Merkle tree is a data structure used to summarize all the transactions in a block into a single hash called the Merkle root. It is used to quickly and efficiently verify that a transaction is included in a block without needing to download the entire block.

6. Differentiate between a public and a private key.
  • Public Key: An address used to receive cryptocurrency. It is shared publicly.
  • Private Key: A secret key used to authorize transactions and access funds. It must be kept private.
7. What is the role of a "node" in a blockchain network?

A node is a computer or device participating in the blockchain network. Nodes perform functions like:

  • Maintaining a copy of the ledger.
  • Validating new transactions and blocks.
  • Participating in the consensus mechanism.
8. What is a genesis block?

The genesis block is the very first block in a blockchain. It is special because it does not have a "previous block" hash to refer to, as it is the beginning of the chain.

9. What is a blockchain explorer?

A blockchain explorer is a web-based tool that allows anyone to view data on a blockchain, including blocks, transactions, and addresses. It provides transparency to the network.

10. Explain the concept of "hashing" in a blockchain context.

Hashing is the process of taking an input of any size and converting it into a fixed-size, seemingly random output string of characters (a hash). In blockchain, it's used to secure each block and link them together to create an immutable ledger.

2. Consensus Mechanisms & Blockchain Types (Q11-20)

Consensus is how the network agrees on the state of the ledger. Understanding the differences between PoW and PoS, as well as the types of blockchains, is essential.

PoW vs. PoS

Proof of Work (PoW): Miners use computational power to solve complex puzzles to validate blocks. (e.g., Bitcoin)

Proof of Stake (PoS): Validators are chosen to create new blocks based on the amount of cryptocurrency they have "staked." (e.g., Ethereum)

11. Explain the Proof of Work (PoW) consensus mechanism.

Miners compete to solve a difficult mathematical puzzle. The first one to find the solution gets to add the next block and is rewarded. The work done is verifiable and expensive, which secures the network from attacks.

12. What are the advantages and disadvantages of PoW?

Advantages: Robust, secure, and has a proven track record.

Disadvantages: Extremely high energy consumption, slow transaction speeds, and a potential for mining centralization.

13. Explain the Proof of Stake (PoS) consensus mechanism.

Validators are selected to validate blocks based on the number of coins they have "staked" as collateral. This method replaces competition with economic commitment.

14. What are the advantages and disadvantages of PoS?

Advantages: Highly energy-efficient, allows for faster transactions, and is more scalable.

Disadvantages: Can lead to a "rich get richer" system, and is less battle-tested than PoW.

15. What is a "fork" in a blockchain?

A fork is a split in the blockchain's single chain of records. It occurs when nodes disagree on the next valid block, resulting in a temporary or permanent divergence of the ledger.

16. What is the difference between a hard fork and a soft fork?

Hard Fork: A permanent divergence from the previous protocol. Older nodes cannot validate the new blocks. Soft Fork: A backward-compatible update to the protocol. Older nodes can still recognize the new blocks as valid.

17. Explain the concept of a "sidechain."

A sidechain is a separate blockchain that runs parallel to the main chain. It is used to offload transactions to a smaller, faster chain to improve scalability, with assets being moved between the two chains.

18. What is the key characteristic of a public blockchain?

Public blockchains are **permissionless**. This means anyone can join, read transactions, and participate in the consensus process without needing approval from a central authority.

19. In what scenarios are private blockchains used?

Private blockchains are typically used within a single organization or business for internal use cases, such as managing a company's internal databases or auditing supply chain logistics, where control and privacy are paramount.

20. What is a consortium blockchain?

A consortium blockchain is a type of permissioned blockchain where a group of pre-selected organizations or entities collectively govern and manage the network. It is a hybrid of public and private blockchains.

3. The Web3 Ecosystem (Q21-30)

This section covers the Web3 ecosystem, including smart contracts, DApps, and cryptocurrencies, which are built on top of the core blockchain layer.

21. What is Web3? How does it differ from Web 2.0?

Web3 is the concept of a decentralized internet. Unlike Web 2.0 where data is owned by corporations, Web3 is "read-write-**own**," giving users ownership of their data and digital assets.

22. What is a smart contract? What is their key purpose?

A smart contract is a self-executing program with the terms of an agreement directly written into code. Their purpose is to automate agreements without the need for a trusted third party.

23. Explain the function of the Ethereum Virtual Machine (EVM).

The EVM is a decentralized, global computer that executes smart contract code. It ensures that every node on the network executes the same code in the same way, guaranteeing a consistent outcome for all transactions.

24. What is a Decentralized Application (DApp)?

A DApp is an application built on a decentralized network, with its backend logic stored in smart contracts on a blockchain. This makes it censorship-resistant and transparent.

25. What is the difference between a coin and a token?

A **coin** (e.g., Bitcoin) is the native cryptocurrency of a blockchain. A **token** (e.g., ERC-20 token) is built on top of an existing blockchain using smart contracts.

26. What is a "Gas Fee" in a smart contract transaction?

A gas fee is the transaction fee paid to validators to compensate them for the computational energy required to execute a smart contract. It is a key part of the network's economics.

27. Define fungible and non-fungible tokens.

Fungible tokens are interchangeable and have the same value (e.g., one Bitcoin is equal to another). Non-fungible tokens (NFTs) are unique and not interchangeable (e.g., a digital artwork).

28. What is a cryptocurrency wallet and what does it store?

A cryptocurrency wallet is a software application that allows users to manage their crypto assets. It does not store the assets themselves, but rather the **private keys** needed to access and control the assets on the blockchain.

29. What is the role of a "validator" in a PoS network?

Validators are nodes that are chosen to create new blocks and validate transactions. They are responsible for securing the network and are rewarded for their participation.

30. Explain the concept of "on-chain" vs. "off-chain" transactions.

An **on-chain** transaction is one that is recorded on the blockchain ledger and validated by the network. An **off-chain** transaction occurs outside of the blockchain, such as on a payment processor, and is not publicly recorded on the main ledger.

4. Critical Security Threats (Q31-40)

This is the heart of the course. Understanding these attacks, especially how they work and how to prevent them, is crucial for a high score. Use the tabs below to explore each major threat category.

31. Explain the 51% Attack. What can an attacker do?

An attacker gains control of over 50% of the network's hashing power. This allows them to double-spend their own coins, block new transactions, and reverse recent transactions. They cannot steal funds from other wallets.

32. What is a Sybil Attack?

A Sybil attack is where a single malicious actor creates multiple fake identities or nodes to gain a disproportionate amount of influence on the network. This can be used to manipulate consensus.

33. Define a "routing attack" on a blockchain.

A routing attack involves an attacker intercepting transaction data as it's being transferred between nodes. This allows them to manipulate or censor transactions before they are confirmed on the blockchain.

34. What is a "timejacking" attack?

A timejacking attack occurs when a hacker manipulates the timestamps of blockchain blocks to mislead other nodes. This can be used to cause forks or trick nodes into accepting fraudulent blocks.

35. What is a "long-range attack" on a PoS network?

An attack specific to Proof of Stake where an attacker who held a private key long ago creates a secret, alternative chain from the beginning of the blockchain's history, then uses it to overtake the main chain.

36. Explain the Re-entrancy Attack.

An attacker's contract repeatedly calls a function on a vulnerable contract before the victim's contract has had a chance to update its state. This allows the attacker to drain the victim contract's funds. It was the cause of the famous DAO hack.

37. What is an Integer Overflow/Underflow vulnerability?

A numerical vulnerability where a variable's value exceeds or falls below its maximum/minimum storage capacity. The number wraps around, leading to unexpected and exploitable results, such as an attacker receiving more tokens than intended.

38. Define a "front-running" attack.

An attacker sees a pending transaction and submits their own transaction with a higher fee so it gets processed first. This allows them to profit from the price change caused by the original transaction.

39. What is a "short address" attack?

This attack exploits smart contracts that don't correctly validate transaction data length. An attacker can send a malformed transaction with a short address, causing the contract to misinterpret the data and potentially send funds to the wrong address.

40. Explain the "timestamp dependency" vulnerability.

A vulnerability where a smart contract's outcome depends on the timestamp provided by the block's miner. Malicious miners can manipulate this timestamp to benefit themselves, for example, by winning a lottery or auction contract.

41. What is phishing in the context of Web3?

Attackers create fake websites that mimic legitimate Web3 services to trick users into revealing their private keys or seed phrases, giving the attacker full control over their funds.

42. Why is it dangerous to share a "seed phrase"?

A seed phrase is a human-readable backup of your private keys. Anyone with your seed phrase can regenerate your keys and access all the funds in your wallet. It should never be shared with anyone.

43. What is a "supply chain" attack in Web3?

An attack where a malicious actor injects harmful code into a widely used software library or component, which is then unknowingly incorporated into many Web3 applications, compromising their security.

44. What is a "malicious airdrop"?

Attackers send free, worthless tokens to a user's wallet. When the user tries to interact with or trade these tokens on a malicious website, they are tricked into signing a transaction that grants permissions to their entire wallet, allowing the attacker to steal funds.

45. Why is it important to verify every transaction you sign in Web3?

Unlike a password, a signed transaction is a cryptographic instruction that cannot be undone. You must verify that the transaction you are signing is what you expect it to be, as a single malicious signature can transfer all your assets to an attacker.

5. Security Best Practices (Q46-50)

To protect against the threats covered, a combination of secure development practices and diligent user habits is required. These are key for both developers and users.

46. What is a smart contract audit?

A smart contract audit is a professional, independent review of a contract's code by security experts to identify vulnerabilities, bugs, and design flaws before deployment to prevent costly exploits.

47. How does the "Checks-Effects-Interactions" pattern prevent Re-entrancy?

It ensures all state changes (`effects`) are completed before any external calls (`interactions`) are made. This prevents an attacker from re-entering the function before the balance has been updated, as the checks would fail on subsequent calls.

48. Why is a hardware wallet considered the most secure way to store cryptocurrency?

A hardware wallet stores your private keys offline on a physical device. This makes them immune to online threats like malware and phishing, as the keys never leave the device and all transactions must be physically confirmed by the user.

49. What is the principle of "least privilege" in smart contract security?

This principle dictates that a smart contract should only be given the minimum set of permissions necessary to perform its functions. Limiting what a contract can do reduces the surface area for potential attacks if a vulnerability is found.

50. Explain the importance of a "fail-safe" or "circuit breaker" function.

A fail-safe is a function in a smart contract that allows a trusted address (like the contract owner) to pause or shut down the contract in an emergency. This can prevent further losses during an active attack.
